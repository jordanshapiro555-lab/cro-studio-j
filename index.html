Process Overview
The full pipeline
📋 Plan▼
📋

Input

  • Strategic business priorities
  • Analysis (audits, analytics & deep dives)
  • CVR not hitting target benchmarks
🧠

Roadmapping

  • Develop a top-line testing strategy for the upcoming cycle
💡

Concepting

  • Develop detailed visuals and scope for test ideas
  • Successive tests and ad-hocs as needed
📑

Prioritization

  • Score concepts on ICE methodology
  • Sort highest-scoring to least
  • Adjust for feasibility

🧪

Creative

  • Develop creative brief
  • Design test concept

Test Configuration

  • Submit items in dev queue for build
🛠️

Development

  • Build test experiences
  • Code review
✅

Tech QA

  • Add QA params to test targeting
  • Tech QA & stakeholder review

🚀

Test Launch

  • Deploy experiment to live environment
🛡️

Monitor

  • Day 1 confirmation of primary metric tracking
  • Continued monitoring of guardrails
⏸️

Pause & Analyze

  • Check for statistical significance at regular intervals
  • Test runs until stat sig or 30 days

Report & Share

  • Win / loss assessment with deep dives as needed
  • Create presentation for results distribution
